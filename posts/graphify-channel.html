graphify 渠道變體版

Telegram / 社群貼文版

`graphify` 最值得追的，不只是它把 repo 變成 knowledge graph，而是它想成為跨 AI coding assistant 的結構化記憶層與導航層。

它現在同時在擴張平台支援、語言支援、MCP / query 能力，也開始碰到大型 repo 的性能瓶頸。這反而是一個很真實的訊號：它已經不只是 skill，而是在往 agent infra 的位置靠近。

Threads / X 短文版

如果你把 `graphify` 看成另一個 repo summary 工具，你會低估它。

它真正有意思的地方，是在搶一個更關鍵的位置：跨 AI coding assistant 的知識圖譜層。

最近最值得看的不是它功能很多，而是它一邊擴張平台、一邊開始撞到大型 repo 的性能與查詢現實。這才是 agent infra 真正的戰場。

部落格導言版

`graphify` 最值得關注的，並不是它又支援了多少平台或多少語言，而是它正在嘗試把大型工作區理解，從「每次重新讀檔」推進到「站在結構化 graph 上導航」。這讓它不再只是 skill，而開始有了基礎設施的味道。
