Telegram / 社群貼文版

最近在看 `NousResearch/hermes-agent`，最大的感覺不是它很紅，而是它已經不太像一般 agent 工具 repo 了。它現在更像一個高速膨脹中的 agent 平台：gateway、cron、skills、memory、subagents、MCP、多 provider、多 channel 全都在一起。真正值得追的地方，是它開始把 agent platform 真正困難的地方暴露出來。

Threads / X 短文版

`hermes-agent` 最值得追的，不只是 9 萬多 stars，而是它已經開始從 agent 工具長成 agent platform。最近最值得看的不是功能列表，而是 gateway、sub-agent、provider abstraction 這些平台摩擦怎麼浮出來。

部落格導言版

如果你只是把 `hermes-agent` 看成另一個 agent 工具，那你很可能低估它了。這個 repo 現在真正有意思的地方，不只是規模，而是它已經在實際碰撞 agent platform 會遇到的那些難題：gateway、session、sub-agent、provider compatibility、長任務 UX 與工具安全。
