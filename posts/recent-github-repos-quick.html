最近在追三個 GitHub repo：

  • Onyx → AI 平台怎麼產品化
  • Harness → 工程治理 / CI-CD 怎麼做穩
  • Humanizer-zh-TW → AI 很會寫之後，文字怎麼修到更像人

這三個 repo 類型差很多，但放在一起看剛好很有意思。因為它們其實對應的是同一條工具鏈上的三個層次：

生成、治理、修整。

Onyx 值得看的是產品演進速度，Harness 值得看的是工程基礎設施的穩定度，而 Humanizer-zh-TW 值得看的是它把「去 AI 味」這件事整理成一套可操作的編輯框架。

一句話：

值得追的 repo，不一定是最大最紅的，而是它有沒有剛好回答一個重要問題。
