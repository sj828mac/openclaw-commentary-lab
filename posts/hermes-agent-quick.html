hermes-agent 短版快評 v1

`hermes-agent` 值得追，不只是因為它 9 萬多 stars，而是因為它已經不太像一般 agent 工具 repo 了。

它現在更像一個高速膨脹中的 agent platform：

  • gateway
  • cron
  • skills
  • memory
  • MCP
  • subagents
  • 多 provider
  • 多 channel

最近最值得看的不是功能有多花，而是平台摩擦開始浮出來了：

  • gateway transcript integrity
  • sub-agent 環境與 polling 問題
  • provider base_url collision
  • long-running task UX

一句話：

Hermes Agent 最有價值的，不只是功能多，而是它正在暴露 agent 平台真正困難的地方。
